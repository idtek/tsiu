Transition from UDT2

If you are familiar with UDT2, you can continue to use UDT3 in the same way, except that you have a new choice now to use partial reliable messaging API. However, bear in mind that UDT2 and UDT3 do not communicate with each other.

Below is a list of changes from UDT2 to UDT3:

  • 1. New messaging API with partial reliability support.
  • 2. Enhanced code base: clearer code structure, less compile options.
  • 3. Various bug fixes.
  • 4. Firewall traversing support (first appeared in UDT 2.2, will be further enhanced in UDT3).

If you never use UDT before, please go ahead with this tutorial.
